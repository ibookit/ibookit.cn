NVIDIA Jetson Nano Developer Kit 新手入门 (2)

安装 Jetson Inference

  • 下载 jetson inference 代码工程，并更新 submodule
  • 执行 cmake，准备编译环境

在 cmake 过程中，需要从 nvidia.app.box.com 下载文件，由于某些不可描述的网络原因，很可能会失败。我的解决办法是申请一个阿里云美西区的 ECS 服务器，在该服务器执行 jetson-inference/CMakePreBuild.sh 下载需要的文件，然后创建一个同可用区的 OSS，将文件通过 ossutil 工具上传到 OSS 中。注意使用 internal endpoint 地址，这样文件 copy 会快很多。再将 ECS 服务器释放，然后通过 ossutil 工具将文件转移到国内 ECS 服务器，同样要注意使用 internal endpoint 地址。最后从国内 ECS 服务器，使用 scp 直接下载文件到 Jetson Nano。 由于所需要的文件压缩后有大约 1G 左右，下载还是需要比较长时间。整个过程顺利的话，估计 1 个小时可以完成。

  • 编译 Jetson Inference

Demo

Jetson Inference 编译成功之后，我们体验两个 Inference 提供的 Demo

  • imagenet-camera: 使用摄像头作为视频源的图像分类
  • detectnet-camera: 使用摄像头作为视频源的物体检测

代码中默认使用 CSI 摄像头作为输入源，代码中 DEFAULT_CAMERA 值为 -1。由于我使用的是一个外接的罗技 C920 摄像头，需要将其值改为 0。

首先修改 jetson-inference/imagenet-camera/imagenet-camera.cpp，找到 DEFAULT_CAMERA，将值改为 0

再修改 jetson-inference/detectnet-camera/detectnet-camera.cpp，同样将 DEFAULT_CAMERA 改为 0

然后回到 build 目录，重新 make，更新可执行文件

Demo 1 - imagenet-camera

执行 imagent-camera，启动 demo 程序。第一次启动时，会有 10 分钟左右时间加载。完成启动后会出现视频窗口，窗口上方不断显示图像分类信息

Demo 2 - detectnet-camera

执行 detectnet-camera, 启动 demo 程序。第一次启动时，会有 10 分钟左右时间加载。完成启动后会出现视频窗口，窗口内移动物体会被矩形阴影标注。这个 Demo 运行过一段时间之后，Nano 竟然过热，自动关机了
