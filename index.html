NVIDIA Jetson Nano Developer Kit 新手入门 (2)

安装 Jetson Inference

  • 下载 jetson inference 代码工程，并更新 submodule
  • 执行 cmake，准备编译环境

在 cmake 过程中，需要从 nvidia.app.box.com 下载文件，由于某些不可描述的网络原因，很可能会失败。我的解决办法是申请一个阿里云美西区的 ECS 服务器，在该服务器执行 jetson-inference/CMakePreBuild.sh 下载需要的文件，然后创建一个同可用区的 OSS，将文件通过 ossutil 工具上传到 OSS 中。注意使用 internal endpoint 地址，这样文件 copy 会快很多。再将 ECS 服务器释放，最后通过 ossutil 工具将文件转移到国内 ECS 服务器，同样要注意使用 internal endpoint 地址。最后从国内 ECS 服务器，使用 scp 直接下载文件到 Jetson Nano。 由于所需要的文件压缩后有大约 1G 左右，下载还是需要比较长时间。整个过程顺利的话，估计 1 个小时可以完成。

  • 编译 Jetson Inference

Demo

Jetson Inference 编译成功之后，我们体验两个 Inference 提供的 Demo

  • imagenet-camera: 使用摄像头作为视频源的图像分类
  • detectnet-camera: 使用摄像头作为视频源的物体检测

代码中默认使用 CSI 摄像头作为输入源，代码中 DEFAULT_CAMERA 值为 -1。由于我使用的是一个外接的逻辑 C920 摄像头，需要将其值改为 0。

首先修改 jetson-inference/imagenet-camera/imagenet-camera.cpp，找到 DEFAULT_CAMERA，将值改为 0

再修改 jetson-inference/detectnet-camera/detectnet-camera.cpp，同样将 DEFAULT_CAMERA 改为 0

然后回到 build 目录，重新 make，更新可执行文件

Demo 1 - imagenet-camera

执行 imagent-camera，启动 demo 程序。第一次启动时，会有 10 分钟左右时间加载。完成启动后会出现视频窗口，窗口上方不断显示图像分类信息

Demo 2 - detectnet-camera

执行 detectnet-camera, 启动 demo 程序。第一次启动时，会有 10 分钟左右时间加载。完成启动后会出现视频窗口，窗口内移动物体会被矩形阴影标注。这个 Demo 运行过一段时间之后，Nano 竟然过热，自动关机了

内容管理产品问题和方法探讨

内容 - 包括什么？

  • 典型应用
    • 在线博客
    • 视频网站
    • 网盘、相册等
  • 内容形式
    • 本地文件：文本、office 文档、图片、pdf、视频 等
    • 外部内容：dropbox、youku、youtube、onedrive 等应用集成

内容如何存放？

  • 本地内容的存储
    • 数据库
    • 原始 File System （包括单机单文件系统、多机多文件系统等情况）
    • 分布式对象存储，如 HDFS 等
    • 云端部署使用云存储，如 S3，OSS 等
    • 通过 Alluxio 连接（adapt）文件系统（包括以上除数据库外的各种类型）

内容生成方式

  • UGC - 用户制作的内容
    • 上传文件
    • 编辑录入文本
    • 编辑录入外部内容信息
  • 系统内部产生的文件，如 Log、录制文件等
  • 本地文件再处理
    • 录制文件转码、生成缩略图等
  • 通过 FTP 等方式批量导入

内容消费方式

  • 文件列表、缩略图展示、下载 （云盘等）
  • 图片浏览
  • 视频播放
  • 文档浏览
  • 在线编辑（例如 Only Office、EtherPad 等）
  • ……

做有内容的产品，或早或晚，你都需要引入 CMS

CMS 典型基础功能需求

  • 内容属性管理
  • 内容 URL 计算
  • 开放 API 接口，支持集成
  • 访问控制，如 authentication key 机制等
  • 多端页面嵌入支持：PC 端，手机端，微信嵌入 （比如提供可以被嵌入的视频播放器）

内容管理页面子系统

  • 根据产品需求不同，是否需要独立的内容管理页面子系统是不确定的
  • 内容页面是系统最核心页面，其主要功能有
    • 内容查看
    • 增、删、改、查

URL 计算

  • 主要针对静态文件为内容的情况讨论，对于类似 Blog、笔记等内容的情况另作考虑，它们多使用数据库作为存储载体
  • 低级方式：本地文件直接映射 URL，如 http://a.b.com/a.txt
  • 普通方式：本地文件，通过服务映射 URL, 如 http://a.b.com/file?id=1234
  • 演进方式：
    • HDFS/OSS 等存储，通过服务映射 URL
    • OSS 等 + CDN （云端部署场景）如，http://cdn.b.com/file/a.txt?auth_key=xxxx

媒体库

  • 功能独立
  • 专注负责各类资源文件的增删改查等管理
  • 与物理文件系统的唯一接口
  • 自有数据库，维护资源文件信息
  • API 接口，方便其他系统集成
  • 资源文件都以 URL 的方式被外部系统访问，比如
    • 图片文件：http://xxxxxx/xxxxx/xxxx.jpg
    • 视频文件：
      • http://xxxxxx/xxxxxx/mp4
      • http://xxxxxx/xxxxx/xxxxx.m3u8
      • rtmp://xxxxx/xxxxx/xxxxxx.flv 等

例子- 视频内容为主的系统的特点

  • VCM 系统存储需求特点
    • 大容量：单一视频文件的大小通常可以达到 G 级*
    • 持续增长：随着使用时间的增加，文件越来越多，占用持续增加
    • 通常不能通过将文件导出的方式来减少占用：文件资源如已经被外部系统所使用，只有当使用者已经不需要时，才可以安全的删除或导出资源文件。通常由于系统之间关系复杂，不易理清文件的使用关系，管理员出于安全和省力方面的考虑，一般不会清理已经存在的文件
    • 高带宽大吞吐量：当应用存在高并发文件访问时，比如视频点播服务，需要文件系统要求高物理带宽已经足够大的吞吐量
    • 高可靠性：文件资源，特别是录制视频，是用户的宝贵资源，需要非常高的可靠性保障，这要求系统带有冗余能力或者足够方便的备份和恢复功能。但是由于系统同时存在文件体积大，数量多的特点，实践经验是提供足够方便的备份和恢复功能的需求是难以实现的

以一个码率为1024kbps的视频为例，一小时长度的大小是 1024x3600 = 约 450MB

物理文件系统选择的目标

  • 选择文件系统时需要衡量的标准
    • 系统是否自带冗余
    • 是否可以方便扩容
    • 是否可以通过增加机器的方式扩大系统整体吞吐量和从带宽角度考虑的负载均衡/分担*
    • 是否带有充分的网络接口
    • 相对低成本

根据部署环境的能力，有可能可以设计上传和下载分别走不同的路径，比如当 OSS 等物理文件系统可以被 CDN 加速时，下载文件可以使用 CDN 加速 URL

例子 - 一个通用文件系统 General File Service （GFS）设计

  • 利用 Alluxio 支持多种类型文件系统，包括对象存储 Aliyun OSS 和 AWS S3，也包括分布式文件系统HDFS
  • OSS 和 S3 可以配置CDN 加速，实现高速文件访问能力，打消系统文件访问IO瓶颈
  • 系统内文件被数据库管理，并且提供 key-value 列表形式的meta data，方便搜索
  • 利用 Alluxio 可以进一步提高文件写操作吞吐量，提升文件上传速度

直播系统微信集成方案

实现微信集成功能的目的

  • 用户身份识别
    • 用户在微信中打开播放页面时，可以获取用户名和头像等信息
      • 可以记录全部观看用户的用户名作为统计信息
      • 可以用作评论功能中的评论者名和头像
      • 直播签到 - 微信内打开观看页面，点击直播签到，输入名字等信息，每个输入与输入时使用的微信号相对应，提高签到有效性
  • 微信用户绑定
  • 当直播系统支持用户系统时，可以支持微信用户和系统用户绑定，从而支持微信登录功能
  • 付费观看
  • 如果直播系统支持付费观看功能，用户可以使用微信支付

准备工作

  • 注册微信公众平台或微信开放平台并通过微信开发者资质认证
    • 如果注册微信公众平台，需要选择服务号，而不能选择订阅号，因为订阅号不能使用用户信息获取等接口
    • 以下是申请微信开放平台的要求，申请公众平台的要求也类似
    • 如果有多个网站应用或移动应用需要集成，或者需要支持 PC 端微信扫码登录功能，则应该使用开放平台，因为公众平台只能设置一个回调域名。开放平台可以创建多个网站应用，每个应用可以设置一个回调域名
  • 微信开放平台注册并开通开发者资质认证
    • https://open.weixin.qq.com
    • 开发者资质认证每年费用为300rmb
    • 需要填写并上传《企业微信开放平台认证申请信息表》（申请表需要公司盖章）
  • 在微信开放平台创建一个网站应用
    • 需要为基成的网站做 ICP 备案
    • 需要填写并上传《微信开放平台网站信息登记表》（登记表需要公司盖章）
    • 设置网站授权回调域为需要集成的网站应用URL，即完成微信登录确认后跳转回的URL
    • 创建成功后，记录网站应用的 AppId 和 App Secret

  • 创建直播活动时，可以设置是否要求观众观看时微信登录

iOS App 发布准备

申请 Apple 开发者账号

申请 D-U-N-S 编号

如果 App 需要在 App Store 上发布，那么建议申请 “以组织身份注册” 的类型。如果选择以组织身份注册，还需要先为您的企业或组织申请 D-U-N-S (邓白氏) 码，请按照 https://developer.apple.com/enroll/duns-lookup/ 的指引完成申请。如果企业资质健全，一般都能申请成功，通常 D-U-N-S 的申请需要几天时间完成。

注册 D-U-N-S 需要提供如下信息以及联系人信息

提交在线申请后，如果申请人是中国境内的组织实体，可能会收到 D & B 的国内代理公司 (上海华夏邓白氏商业信息咨询有限公司) 发来的邮件，要求签署邮件附件中的确认知悉函，并提供以下信息

  • 公司清晰的营业执照正本（或副本）照片和公司公章刻字面的实物照片（如是事业单位请提供事业单位法人证）
  • 公司中文名称（工商注册的中文名，如有更名情况，请同时提供更名前和更名后的名称）、英文名（如无正式的英文名按字面翻译）
  • 中文办公地址、英文办公地址
  • 邮编
  • 企业类型
  • 是否是独立企业（公司有无分公司或办事处，只需填有或无）
  • 公司英文简称或英文商标（如果没有可以填无）
  • 主营业务（一个最主要的业务）
  • 员工人数（请尽量精确,例:10人；不要填区间,例：10-20人）
  • 本项目联系人（务必填写申请公司的本项目联系人），包括姓名、 性别、 职务和电话（公司座机和联系人手机，所提供座机和办公地址保持一致）

完成苹果开发者注册

以组织身份注册苹果开发者，需要在获得 D-U-N-S 编号后，才可以继续完成注册，需要提供如下信息

苹果开发者的注册地址是 https://developer.apple.com/programs/，网站上有详细的流程介绍。

开发者账号注册好之后，才可以进行后续步骤。

常用网站

  • Apple 开发者网站 - https://developer.apple.com， 管理 App ID，签名证书，Provisioning Profile 等
  • App Store Connect 网站 - https://appstoreconnect.apple.com，管理 App，Test Flight 测试版本，用户等
  • Apple ID 管理网站 - https://appleid.apple.com/cn，管理开发者的 Apple ID，需要在里边创建 App specific password，在上传 ipa 文件的命令中需要使用

创建 App ID

在 https://developer.apple.com 网站为 App 创建一个 App ID，创建 App ID 是为了创建 Bundle ID （PRODUCT_BUNDLE_IDENTIFIER）。如果在 Xcode 工程中使用 xcconfig，则 release build 的 xcconfig 中可以设置 PRODUCT_BUNDLE_IDENTIFIER = "创建 App ID 时指定的 Bundle ID"。

创建 iOS Distribution 证书

这个证书用于 Xcode 编译过程中为生成的 ipa 文件签名，如果还没有创建该证书，则可以使用 Xcode 创建该证书。创建的方法是进入到 Xcode > Preferences > Accounts > Manage Certificates…, 点 + 号，选择 iOS Distribution。

创建出来后，可以在主机的 Keychain Access 应用中将该证书导出为一个 .p12 文件备用，一般需要将这个证书再安装到编译服务器上去。

创建 iOS Provisioning Profiles (Distribution)

在 https://developer.apple.com 网站为 App 创建一个 iOS Provisioning Profiles (Distribution)，这个文件也是在 Xcode 编译时使用。创建时，需要选择已经创建好的 App ID，以及 iOS Distribution 证书。创建好的 iOS Provisioning Profiles (Distribution) 文件可以下载下来，安装到编译服务器上。

配置 release xcconfig 文件

建议在工程中使用 release xcconfig 文件，可以在该文件中配置好以下几项配置。在工程的 Build Settings 中将相应的属性设置为对应的参数。如将 Provisioning Profile > Release 设置为 ${PROVISIONING_PROFILE_SPECIFIER}，将 Packaging > Product Bundle Identifier > Release 设置为 ${PRODUCT_BUNDLE_IDENTIFIER} 以此类推。

在 App Store Connect 创建应用

在 App Store Connect 网站中创建一个新的 iOS App，创建应用时选择之前创建的 Bundle ID。只有创建 App 后才可以上传改应用的 ipa 文件。

编译 Release 版本应用 ipa 文件并将其上传到 App Store Connect

建议在编译服务器通过命令行方式编译用于发布的 ipa 文件，然后通过命令行进行上传。可以使用 Jenkins 等 CI/CD 工具，将这一过程自动化。编译命令和上传命令示例如下

苹果现在要求 altool 命令中不能再使用用户的登录密码，而需要使用 App Specific Password，这个密码需要在 Apple ID 管理网站中创建

通过 Test Flight 安装 App 测试

Test Flight 是苹果推荐的 App 发布前的测试和 Beta 手段， ipa 上传到 App Store Connect 后，测试人员可以通过 Test Flight 安装新上传的 App，进行必要的测试。有关 Test Flight 的使用方法可以参考 https://developer.apple.com/testflight/

提交审核

新版本应用通过测试后，可以将其提交审核。在 https://appstoreconnect.apple.com 网站上提供必要的信息，然后提交审核即可。苹果一般几天后开始审核，如果审核成功，则 App 可以发布到 App Store。如果审核失败，苹果会提供反馈信息，说明审核失败的原因，需要根据反馈进行必要的修改或者同审核人员进行必要的沟通。可以参考苹果文档 https://help.apple.com/app-store-connect/#/dev34e9bbb5a 查看更详细的说明。

苹果现在要求所有应用需要提供《隐私协议》链接，需要将准备好的《隐私协议》放到公司网站后，将地址链接填入 App 审核申请中对应的空格。

另外，有时苹果要求提供 App 使用的 Demo 视频，建议录制一个 Demo 视频，在申请表中提供视频的地址链接。
